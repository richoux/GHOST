60  /*
61  Why having both domain and indexes vectors?
62 
63  The domain vector contains integers modelling possible values of a variable. Such values can be
64  {7, -1, 3}. Then, your domain can be not canonically ordered and have 'holes', ie, non-contiguous integers.
65  Thus, it can be more convient for Variable objects to handle the index of their value in the domain rather
66  than their value itself. Indeed, taking the next value in the domain is just incrementing the current index,
67  rather than searching for the next possible value in the domain.
68  However, it is necessary to know which index is associated to a value, for instance for setting a new value
69  (in fact, a new index) to a variable. We had then two choices: searching for such an index in the domain vector
70  or storing all indexes in another vector. For speed sake, we chose this second option.
71  */
